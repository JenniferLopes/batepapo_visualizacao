TidyTuesday: como descobrir

O TidyTuesday é uma iniciativa semanal da comunidade de R criada e mantida por Jon Harmon e colaboradores da Data Science Learning Community.

Todo ano começa e termina com um dataset especial. O restante do ano tem um dataset novo por semana.

Repositório oficial: github.com/rfordatascience/tidytuesday

TidyTuesday: estrutura prática

Como funciona:

  • Dataset novo toda semana
  • Tema variado: saúde, clima, cultura, esporte, economia
  • Sem resposta certa. A análise é livre
  • Compartilhamento público no GitHub e redes sociais

Como participar:

  • Acesse o repositório e baixe o dataset da semana
  • Faça sua análise em R ou Python
  • Compartilhe no GitHub com o código
  • Use a hashtag #TidyTuesday no LinkedIn e no Mastodon

O que muda quando você participa de desafios

  • Você aprende a trabalhar com dados que não escolheu
  • Desenvolve velocidade de análise
  • Cria o hábito de documentar e publicar
  • Ganha repertório visual com as soluções de outras pessoas

Tip

Participar de um desafio por semana por três meses é mais formativo do que assistir a horas de curso sem praticar.

Ferramentas e prática

TidyTuesday como ferramenta de evolução prática

O TidyTuesday não é só um desafio. É um treino semanal com dados reais, sem pressão de entrega e com audiência real.

O que você desenvolve ao longo do tempo:

  • Velocidade de limpeza e manipulação de dados
  • Repertório de tipos de gráficos e quando usá-los
  • Hábito de documentar decisões analíticas
  • Portfólio público com histórico de evolução visível

Outros desafios e plataformas

  • #30DayChartChallenge: um gráfico por dia durante 30 dias em abril. Tema diferente a cada dia.

  • #30DayMapChallenge: variação focada em mapas durante novembro.

  • Makeover Monday: reformular visualizações existentes com dados reais.

  • Observable Plot Gallery: exploração livre com a biblioteca Observable.

R e Python no contexto de visualização

R

  • ggplot2: padrão de mercado para visualização estática
  • plotly: interatividade com sintaxe familiar
  • gganimate: animações sobre gráficos ggplot2
  • ggridges, ggbeeswarm, ggdist: extensões especializadas
  • Quarto para publicação integrada

Python

  • matplotlib: base de tudo, altamente configurável
  • seaborn: API de alto nível sobre matplotlib
  • plotly: interatividade e dashboards
  • altair: gramática declarativa de visualização
  • bokeh: visualizações interativas para web

Ferramentas mais utilizadas no dia a dia

  • ggplot2: análise exploratória e comunicação final
  • Quarto: documento + código + visualização publicados juntos
  • GitHub: versionamento e portfólio
  • Flourish: visualizações interativas sem código
  • Datawrapper: gráficos e mapas para publicação rápida
  • R Graph Gallery: referência de código por tipo de gráfico

Como sair do básico e evoluir na prática

Três movimentos que mudam o nível:

  1. Copiar com consciência: reproduzir o código de quem você admira e entender cada linha
  2. Publicar antes de estar pronto: o feedback real acelera mais do que a revisão solitária
  3. Documentar as decisões: escrever por que você escolheu aquele gráfico ensina mais do que fazer o gráfico

Bloco 5

Construção prática de visualizações

Como começar hoje

Não comece pelo gráfico. Comece pela pergunta.

  1. Qual é a pergunta que você quer responder?
  2. Qual variável responde essa pergunta?
  3. Quantas dimensões estão envolvidas?
  4. Qual tipo de gráfico comunica melhor essa relação?
  5. Quem vai ler esse gráfico?

Important

Um gráfico tecnicamente correto que não comunica para o público certo é um gráfico que falhou.

Escolha de ferramentas: ggplot2 no R

A gramática de gráficos do ggplot2 é baseada em camadas:

library(ggplot2)

ggplot(dados, aes(x = variavel_x, y = variavel_y)) +
  geom_point() +
  geom_smooth(method = "lm") +
  labs(
    title    = "Título informativo",
    subtitle = "Contexto adicional",
    x        = "Rótulo do eixo x",
    y        = "Rótulo do eixo y",
    caption  = "Fonte dos dados") +
  theme_minimal()

Separar análise de visualização

Análise exploratória

  • Gráficos rápidos e sujos
  • Objetivo: entender os dados
  • Não precisa ser bonito
  • plot(), hist(), ggplot() básico
  • Resultado: insight

Visualização comunicativa

  • Gráficos refinados e documentados
  • Objetivo: comunicar para outra pessoa
  • Cada elemento tem uma razão de existir
  • Tema, paleta, rótulos e fontes deliberados
  • Resultado: decisão ou compreensão

Como contar uma história com dados

Uma visualização que conta uma história tem:

  • Contexto: quem são os dados, de onde vieram, qual o período
  • Tensão: o que é surpreendente, contraditório ou relevante
  • Resolução: o que os dados indicam e qual é a implicação prática

O título do gráfico não deve descrever o que está representado. Deve comunicar a conclusão.

Tip

Título fraco: “Vendas por região em 2023”

Título forte: “Norte concentra 40% das vendas, mas recebe apenas 15% do investimento”

Publicação: GitHub e LinkedIn

GitHub:

  • Repositório público com código, dados e README documentado
  • Histórico de commits como registro de evolução
  • Referência que recrutadores e colaboradores consultam

LinkedIn:

  • Publique o gráfico com a pergunta que ele responde
  • Mostre o processo, não só o resultado final
  • Três parágrafos: contexto, o que os dados mostram, o que isso significa

Importância de mostrar o processo

O resultado final esconde o aprendizado. O processo expõe a competência.

Mostre:

  • O dataset bruto antes da limpeza
  • As versões descartadas do gráfico e por que foram descartadas
  • Os erros que ocorreram e como foram resolvidos
  • As decisões de design e a justificativa de cada uma

Quem contrata quer ver como você pensa. Não só o que você entrega.

Bloco 6

Fechamento e direcionamento

Importância de ter um portfólio

Portfólio não é currículo com imagens.

É evidência de como você pensa, resolve problemas e comunica resultados.

Um projeto publicado com uma decisão bem justificada vale mais do que dez projetos guardados esperando o momento certo.

Consistência x perfeição

Esperando a perfeição:

  • Ninguém vê
  • Não gera oportunidade
  • Fica desatualizado
  • Aumenta a paralisia

Publicando com consistência:

  • Mostra como você pensa
  • Abre portas concretas
  • Evolui em público
  • Gera conexões e feedback real

Important

O portfólio que ninguém vê não abre nenhuma porta.

Próximos passos para quem está começando

  1. Escolha um dataset do TidyTuesday desta semana
  2. Faça uma análise exploratória com ggplot2
  3. Escolha um gráfico e refine até ele comunicar uma conclusão clara
  4. Publique no GitHub com README documentado
  5. Escreva um post no LinkedIn descrevendo o processo

Não espere estar pronta. O aprendizado acontece publicando.

Mensagem final

Aprender visualização de dados tem dois componentes insubstituíveis:

Prática: fazer gráficos toda semana, independente de serem perfeitos.

Exposição: publicar o que você faz e receber feedback real.

Nenhum curso substitui os dois juntos.
